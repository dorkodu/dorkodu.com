Introducing Sage

Sage is a query-based data exchange protocol for APIs.

Sage aims to simplify the communication for data interactions between different layers of software, and designed especially for APIs.

Sage is open data.

Get Started Learn More

What is Sage?

Sage is an open protocol for designing and developing better APIs, which is a specification about how to query/response for data and a runtime capable of executing those queries with your existing business logic.

Sage provides a simple & lightweight but also efficient, expressive and intuitive way for describing the data graph in your API, giving clients the power to interact with your API exactly how they need to —performing data retrieval or action call— makes evolving and documenting your APIs easier.

We created Sage for catalyzing the Web 3.0 revolution.

We develop Sage as open-source on GitHub

Simplify data exchange.

Data exchange should be simple, lightweight and efficient. That’s what Sage is all about. Sage helps you building more powerful open data APIs.

Client-specified interactions
Sage queries are specified detailedly, in the client rather than the server. Clients determine the interactions—data fetching or action calls—and does exactly what a client wants and no more.
On your own business logic, with your own data
Sage creates a uniform API across your entire application. Build Sage APIs that leverage your existing data and code–business logic–with a Sage runtime.
Fewer bytes and roundtrips
Improve the experience of your apps by only retrieving the data you're actually using, and reduce the number of roundtrips to fetch all of the resources for a particular view.

Efficiency at Scale.

Sage is super efficient. Its power comes from its simplicity and lightness, which means it is optimized for you as human (to create fast) and the computer (to run fast). Because Sage is so efficient, that makes it much easier to scale.

Rapid and "Lucid" Development
Iterate quickly on apps without wasting your time for new API endpoints. Sage simplifies data exchange, so you can develop more consciously and faster by getting the data exactly as how you want.
Save resources
More efficiency means you need less processing power and fewer servers. Avoid unneeded maintenance, use your time to focus on more meaningful things.
